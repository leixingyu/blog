How to Organize Custom Python Library

2021-08-14 general pipeline, python, style

To create a physics constraint there must at least have two components, can be none which is the scene root

The Base can be considered as static and the other component has simulated physics
The component with simulated physics on should have an appropriate mass

The constraint Disabled collision should be enabled if possible to avoid collision between the base versus the simulated component
Apply linear limits and angular limits accordingly Usually if you want the movement on certain axes either rotation or translation toggle on free or limited motion

Limited motion allows you to specify a limit range also enabling you to turn on soft constraint
The sub constraint being soft means that you can also tweak the stiffness and the damping of the soft constraint to mimic a spring behavior
Thereâ€™s also a angular breakable Or linear breakable which can be helpful to break the constraint when the force reached a certain threshold

To allow the simulated component return back or maintain its initial transform enabling motors such as linear motor or angular motor is highly recommended The target should be kept at zero if the initial position is the desired target the drives should match the movement along certain axes and that the strength is how strong the motor forces to drive the simulated component back to the target
The target velocity will help reduce the wobbleness caused by the motor fighting the gravity

the motor force is determined by both the strength and multiply the mass/weight of the simulated component

not starting with non-neutral position

e.g. initial pulled back spring board that will retrun to neutral position pull the simulated component in the viewport and apply the counter offset value which will put the component back to neutural

if the neutral is 0, 0, 120 pull back is 0, 0, 60

  1. in the viewport set it to 0, 0, 60
  2. then in the linear motor set it to 0, 0, -60 (60-120)
  3. this will make the spring board launch to the neutral position when starting as pulled back

this can be done in blueprint via set linear position target on the constraint

sleep veolicity threshold and wake rigid body

you can also use this to dynamically change the target, for instance you need to make a radial switch level that can flip flop between 2 target state

you can also do a target velocity only (meaning no target position/rotation), this will be able to apply a continuous movement/velocity to keep it moving, good for inifinite rototaing door

The base can be hidden or set to not visible but it has to exist because the physics constraint needs two components to work you can also disable collision on the base by setting the collision preset to query only and ignore everything
