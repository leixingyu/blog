UE5 Cinematic Guide (Part.2 Asset and Animation Transfer)

2022-08-13 cinematic workflow maya, pipeline, unreal

Introduction

How do we transfer assets and animation to Unreal needed for cinematic? For now we’ll stick to .fbx until .usd becomes a norm. For each component of a cinematic, we want to set it up nicely so the data can be correctly presented in Unreal.

However, do note there are limitations in .fbx and differences between DCCs, certain things just can’t be transferred through .fbx and certain things can’t be represented in Unreal. Always test your workflow before going into full scale production.

General

The important thing to keep in mind is that consistency is king, we want to have consistent world unit, identify coordinate system for each DCC, and maintain a hierarchy standard for assets across DCCs.

Certain workflow involves animating or placing components within Unreal instead of importing outside the engine which is totally fine.

Static Mesh Animation

If a mesh is animated in any capacity, even if it’s just translating or rotating as a whole without any deformation, it needs to be skinned with joint(s). Consider this as a common standard.

Rigging in Game vs Animation

A complex animation rig can’t translate 1 to 1 to game engine, the rigging phase needs to be planned to accommodate the limitation of the engine.

Process: Clean-up the rig, such as grouping/un-grouping, get rid of namespace and unwanted components such as controllers and constraints.

Skeletal Mesh Animation

Whatever the character skeleton rig hierarchy in the .fbx looks like, the exported animation should match that structure 1-to-1. Otherwise, there will be a weird transform offset. Also, the root of the rig/skeleton, whether it’s a transform group or a root joint, should not have animation/keyed data on it, this is because it is used to offset world transform (and keyed transform can’t do that)

for example: the character rig hierarchy could look like

  • parent transform [no key]
    • offset transform [no key]
      • hip joint
        • right leg joint
        • left leg joint

or as simple as

  • root joint [no key]
    • hip joint
      • left leg joint
      • right leg joint

Export

the keyframes are usually on controllers, so we have to bake the joint animation, Select the joint hierarchy and bake simulation, now we can get rid of the controllers and constraints.

select the joint hierarchy and export

Blendshape

additional steps needs to be done to export blendshape animation

  1. find all the meshes with blendshape node attached
  2. traverse in the node editor and locate the blendshape node
  3. bake simulation on the blendshape node
  4. select not only the joint hierarchy, but also the meshes with blendshape and export

Camera Animation

you can’t export and import a maya camera asset. Unreal has it’s own native Camera type which is not exactly 1 to 1 match to Maya’s Camera.

What you can do is create a CameraActor in Unreal and import Maya camera’s animation and attribute for cinematic. (a different story for gameplay)

Export

the camera node has to be on the scene root, with no transform node on top. Otherwise, the animation won’t be imported
