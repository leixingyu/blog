General File Structure

2021-09-17 general pipeline

Settings

Game Mode Blueprint sets:

  • PlayerController Blueprint Class
  • Default Pawn Class: our custom character blueprint

Character Blueprint

Base class is the “Pawn” class: Pawn determines what the player or AI entity looks like visually and the physical location, rotation, etc, also how it interacts with the world in terms of collisions and other physical interactions

“Character” is a special type of “Pawn”, with the feature built-in by Unreal such as walk, fly, crouch, and etc.

Mesh

In the class default’s “Animation” section, set “Animation Mode” to “Use Animation Blueprint” and set “Anim Class” to our custom animation blueprint

Player Controller

Controllers are non-physical Actors that can possess a Pawn (or Pawn-derived class like Character) to control its actions.

This in my opinion isn’t explained well in all the tutorial videos, since you can access the “controller” from the character blueprint (since it possessed the character at runtime to be able to control it), you can almost think of this Player Controller as an invisible sub-component of the character (but it can be added or removed at anytime).

https://youtu.be/p-wZoAUzgNk?si=fNEuixh8QVppiW9J

Camera Setup

Fixed Character Camera (Unreal default third-person camera): camera yaw will turn character

Semi-locked Character Camera (): camera yaw won’t turn character, but directional movement will turn character

Flexible Character Camera (The Last of Us Camera): camera yaw and movement won’t turn character, animation blueprint will turn character

  • On the Character BP’s class default, toggle off “Use Controller Rotation Yaw”: this prevents the character to update yaw rotation to match the playerController

  • On the “Spring Arm” component, toggle on “Use Pawn Control Rotation”: this makes the spring arm (and the child camera component) to update yaw rotation to match the playerController rotation

  • (Optional): see more in movement component

  • The Camera Boom is the pivot point of the camera movement, we want to move the camera itself for offsetting

Movement Component

Depending on the desired outcome, the Character Movement (rotation settings) needs to be tweaked:

  • “orient rotation to movement” may need to be toggled off, otherwise the character will rotate as your have turn movement (which is good if you want a semi-locked character Camera).
  • “use controller desired rotation” will make the character rotate slowly towards the controller’s rotation
  • at last, we can use a custom blueprint logic to mimic the “use controller desired rotation” only when the character is moving using the Actors’ Forward Vector and slowly interpolate (RInterpolate To) from Actor Rotation to Control Rotation (Yaw only) when moved. source

Animation Blueprint

Animation is what makes the Mesh animated within the Character BP, it’s used to create the illusion that the animation is driving the Pawn capsule, but it’s actually in reverse: the movement drives the animation triggering different state and play different animation clips

Event Graph

Which is where all the initialization and update calculation happens to update user defined variable which drives the transition logic:

  • example: during initialization event, grab the parent character’s (i.e. the character blueprint instance) movement component to access the character’s movement direction and speed.
    • note: you can think of animation almost as an additive layer that gets played on the character based on certain logic affected by the Character BP’s movement, but not vice versa

Animation Graph

Where the state-machine is hosted and animation sequence assets are specified

State Alias:

If multiple states have the same transition logic to another state:

  • e.g. all below uses the same ‘isFalling’ transition logic
    • Locomotion - Fall Loop
    • Jump - Fall Loop
    • Crouch - Fall Loop

We can create an intermediate State known as ‘state alias’, which is like a central hub; this State Alias can be named something like ‘to Fall Loop’ to be more comprehensive, and then check the “Locomotion”, “Jump”, “Crouch” option so we know these three state shares this State Alias.

Next we can substitute the three transitions with one transition:

  • to Fall Loop - Fall Loop (using the same ‘isFalling’ transition logic)

Cached Pose:

This is almost for organization purpose, for cleaner animation graph, we can reference saved pose node for convinence access.

Post-process Animation Blueprint

is a special Blueprint assigned to the Skeletal Mesh asset to do anything after the animation is applied

  • example: pose-based deformation
  • able to preview the effect directly in animation sequence preview window once the post-process BP is applied to the corresponding SKM
