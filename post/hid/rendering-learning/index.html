Learning Rendering Basics

2021-02-18 learning log graphic programming Comments

Rendering Software

The rendering software/engine (or renderer) is the module/application
that is reasonable for generating the graphical output.
Basically the job of a rendering engine is to convert the applications
internal model into a series of pixel brightness’s that can be displayed
by a monitor.

Photorealistic offline rendering

Renderer (such as Arnold, V-ray, RenderMan)
that uses ray tracing to achieve realistic rendering;
frames are rendered to image format and displayed later.

real-time hardware-assisted rendering

Renderer using a traditional rendering pipeline
(APIs such as OpenGL or DirectX) with rasterization,
not designed to perform raytracing.

Rendering APIs

Application programming interface or libraries are designed to handle complex math,
drawing operation and talks to 3D graphics hardware for video games and
3D real-time viewport in varies dcc application.
It is typically used to interact with GPU to achieve hardware-accelerated
rendering.

  • OpenGL: is a cross-language, cross-platform API for rendering 2D and 3D vector graphics.

  • DirectX: on the other hand, only runs on Microsoft platform (Windows and XBox).

Video game

A lot of game engines have a ‘graphics middleware’ layer that can make calls
to OpenGL or Direct 3D functions depending on what platform you’re on.

Viewport

3D Viewport render allows for quick preview (lighting, shadows, material,
animation…) as opposed to final render.

Viewport render are fast but not guarantee to playback in real-time
if the scene is too complex, thus the need of using playblast.

Hardware acceleration: is the use of computer hardware designed to
perform specific functions more efficiently when compared to software
running on a general-purpose central processing unit (CPU).
Any transformation of data that can be calculated in software running on a
generic CPU can also be calculated in custom-made hardware, or in some mix of both.

General Purpose GPU rendering: As NVIDIA started making
the GPU’s more programmable, it found that it could use the massive
computational power in GPU’s to do general purpose computing.
This led to the development of CUDA (Compute Unified Device Architecture).
This enables programmers to write software that can run on the GPU
instead of the CPU. The benefit of doing this is in parallelization.
A modern GPU might have 480 cores while a CPU has 8. So if you can
make your program more parallel, you’ll potentially get quite good
speed out of using a GPU.
